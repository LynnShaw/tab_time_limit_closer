# Tab Time Limiter

一款 Chrome 浏览器扩展，为指定网站设置使用时间限额，超出配额后自动关闭标签页，帮助你减少分心、提高专注力。

## 功能

- **按域名限制使用时长**：为任意网站设置每个时间窗口内的最大使用分钟数
- **灵活的计时窗口**：可自定义窗口长度（如每 4 小时、每 24 小时），窗口结束后配额自动重置
- **实时进度展示**：弹窗中以进度条 + 颜色状态实时显示每个网站的剩余时间
- **2 分钟宽限期**：超出配额后先推送桌面通知，2 分钟后再关闭相关标签页
- **快速预设**：内置 30 分/天、1 小时/天、2 小时/天、30 分/4 小时等常用规则，一键填入
- **事件驱动计时**：切换标签、切换窗口、页面跳转时精确开始/结束计时，不依赖轮询，误差极小
- **Service Worker 崩溃恢复**：Chrome 杀死后台 Service Worker 后，下次唤醒时自动补算遗漏时间，并恢复待关闭标签页的倒计时

## 安装

本扩展尚未上架 Chrome 应用商店，需以开发者模式手动加载。

1. 下载或克隆本仓库到本地
2. 在 Chrome 地址栏输入 `chrome://extensions/` 并回车
3. 打开右上角的**开发者模式**开关
4. 点击**加载已解压的扩展程序**，选择本项目根目录
5. 扩展图标出现在工具栏后即安装成功

> 如果图标未显示，点击工具栏右侧的拼图图标（扩展管理），将 **Tab Time Limiter** 固定到工具栏。

## 使用方法

### 添加限制规则

1. 点击工具栏中的扩展图标，打开弹窗
2. 点击右上角 **+** 按钮展开表单
3. 填写**网站域名**（如 `youtube.com`）、**时间限额（分钟）** 和**计时窗口（小时）**，或点击快速预设
4. 点击**保存**

### 编辑 / 删除规则

在弹窗的网站列表中，每张卡片右侧有编辑（✏️）和删除（🗑️）按钮。编辑规则后使用记录会自动清零。

### 状态说明

| 徽章颜色 | 含义 |
|---------|------|
| 绿色·正常 | 使用量低于 60% |
| 黄色·即将用完 | 使用量达到 85% 以上 |
| 红色·已封锁 | 配额用尽，标签页将被关闭 |

进度条旁边的闪烁绿点表示当前标签页正在计时中。

## 文件结构

```

├── manifest.json      # 扩展清单（Manifest V3）
├── background.js      # Service Worker：计时核心、关闭逻辑、通知
├── popup.html         # 弹窗 HTML 骨架
├── popup.js           # 弹窗交互与渲染逻辑
├── popup.css          # 弹窗样式

```

## 权限说明

| 权限 | 用途 |
|------|------|
| `tabs` | 监听标签页切换、导航和关闭事件 |
| `storage` | 持久化网站规则与使用记录 |
| `alarms` | 每 30 秒触发一次后台检查（Chrome 允许的最小间隔） |
| `notifications` | 配额用尽时推送桌面提醒 |

## 技术实现

- **Manifest V3 Service Worker**：后台逻辑运行在 Service Worker 中，Chrome 可能随时将其挂起。扩展通过将活跃会话和待关闭队列持久化到 `chrome.storage.local` 来应对这一情况，重启后自动补算遗漏时长并继续执行宽限期后的关页动作。
- **事件驱动计时**：`tabs.onActivated`、`tabs.onUpdated`、`windows.onFocusChanged`、`tabs.onRemoved` 四个事件联动，精确记录每次会话的起止时间。
- **URL 匹配**：通过 hostname 前缀匹配，`youtube.com` 可同时覆盖 `www.youtube.com`、`m.youtube.com` 等所有子域名。
- **窗口重置**：计时窗口基于首次访问时间滚动计算，窗口到期后配额自动归零。
